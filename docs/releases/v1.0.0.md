# VulnTrack API v1.0.0

The first stable release delivers the complete CRUD baseline for validated
information-security findings.

## Highlights

- FastAPI health and finding endpoints with generated OpenAPI documentation.
- Strict Pydantic validation for severity, status, CVSS, text, and updates.
- Persistent SQLite repository using parameterized statements.
- 59 unit/integration/smoke tests with an enforced 80% coverage floor.
- Sequential lint, test, build, security, and release jobs.
- Bandit, pip-audit, detect-secrets, least-privilege CI, and versioned Git hooks.
- Editable engineering report, platform comparison, and evidence register.

## Install the artifact

Download the wheel attached to this release, then run:

```bash
python -m pip install vulntrack_api-1.0.0-py3-none-any.whl
VULNTRACK_DATABASE=./findings.sqlite3 \
  python -m uvicorn vulntrack.main:app --host 127.0.0.1 --port 8000
```

The release also includes a source distribution. Both files are produced by
the same workflow run that passes lint, tests, coverage, build, and security.

## Known limits

- Filtering and severity statistics are planned for v1.1.0.
- SQLite is suitable for the assignment and a single service instance, not a
  horizontally scaled production deployment.
- Independent approval and enforced `main` branch rules require an external
  collaborator/repository setting and are not claimed by this release.
