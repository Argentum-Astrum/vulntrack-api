# VulnTrack API v1.1.0

This backward-compatible minor release adds reporting operations to the stable
CRUD baseline.

## Highlights

- `GET /findings?severity=...&status=...` supports either filter or both.
- Invalid filter enums return `422` instead of an ambiguous empty collection.
- `GET /findings/statistics` returns the total and all four severity counts.
- Static parameterized SQLite filtering keeps user input out of SQL syntax.
- 65 unit/integration/smoke tests cover the release at 97.67% branch coverage
  before the release run.

## Compatibility

Existing v1.0.0 CRUD requests and response models are unchanged. New response
fields appear only on the new statistics endpoint.

## Install the artifact

Download the attached wheel, then run:

```bash
python -m pip install vulntrack_api-1.1.0-py3-none-any.whl
VULNTRACK_DATABASE=./findings.sqlite3 \
  python -m uvicorn vulntrack.main:app --host 127.0.0.1 --port 8000
```

The wheel and source distribution are produced and attached only after lint,
tests, coverage, build, Bandit, pip-audit, and detect-secrets pass in the same
workflow run.
